1) Project Goal (Simple Explanation)

This project is a cinematic Prompt Bank web app that lets user browse prompts like a streaming platform, then directly send selected prompt content into existing Python software PromptTyper.

Core idea: User should not manually edit code every time. They can add prompt (title + text, optional image), search, filter, preview, and push to PromptTyper content box for shortcut mapping.

2) End-to-End User Flow

  1. Open web app -> cinematic intro loads (image preload + intro sound segment).
  2. User sees hero slider with big posters and dot navigation.
  3. User searches prompt or browses categories / quick filters.
  4. User opens prompt detail -> can copy or click Save to PromptTyper.
  5. App sends prompt to local PromptTyper bridge API.
  6. User is redirected/opened in PromptTyper manage flow and sets shortcut there.

4) Key Features You Should Explain

A. Cinematic Intro

  • Intro overlay preloads images to avoid blank flashes.
  • Audio segment is controlled (start at 5s, stop at 10s).
  • Fade-in and fade-out applied for smooth sound transition.

B. Netflix-style Hero Carousel

  • Large poster cards with center focus effect.
  • Auto-slide + manual prev/next arrows + active dots.
  • Stable slide transform logic to avoid progressive right drift bug.

C. Prompt Discovery

  • Search on title/category/tagline/prompt content.
  • Category filters + quick filters: All, Pinned, Favorites, Recent.
  • Prompt detail view with copy action and screenshot previews.

D. Admin: Add Prompt (Main Feature)

  • Required: title + prompt text.
  • Optional: image (if present, can be featured in top slider).
  • Edit/Delete support from saved prompts list.
  • Image preview before save.

E. PromptTyper Integration

  • Top status: active / app not running.
  • Save to PromptTyper sends selected content to Python tool through local API.
  • This avoids direct auto-shortcut creation; user sets shortcut safely in PromptTyper manage section.

9) Viva Q&A (Ready Answers)

Q: Why local bridge API instead of direct DB in browser?
A: We need integration with existing Python software (PromptTyper). Local bridge allows controlled read/write and direct handoff.

Q: Why title+prompt mandatory and image optional?
A: Core text prompt must always exist. Image is only for visual presentation and slider listing.

Q: How do you handle browser autoplay blocks?
A: Autoplay is attempted first, and if blocked we show tap fallback. This aligns with browser media security policy.

Q: How is user data persisted?
A: Prompt library through bridge endpoint; quick user flags like pinned/favorite/recent in localStorage.

Q: Biggest improvement if next version?
A: Add authentication + multi-user profile + cloud sync + searchable tags and analytics dashboard.
